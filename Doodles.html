Before working on this website, I had never coded before, never really done anything in design and had never (or rarely) used many of the tools, technologies of the profession - HTML and CSS, for example. It was also an introduction to Design Theory proper. A lot of the knowledge I consumed in this module, I only really started to understand or use properly 2 or maybe 3 modules later.

This Module introduced me using Mood boards and Style tiles as a method of design and development

This was the first time I attempted to use CSS Grid Layout and although I had some level of success, I'd definitely say I exhibited far greater understanding of the tool in future modules.

I was also attempting to make use of the different Design Principles we had learned this module, such as use of visual contrasts to make certain parts stand out more

PERSONAL OBSERVATIONS:

My biggest take away from this module was the use of folder structure, file paths and file naming in structuring user interfaces. Learning about the information architecture of content so early put me in good stead for the rest of the first semester. It really helped me to properly organize my work, which makes it far easier for me to make changes should I need to.
